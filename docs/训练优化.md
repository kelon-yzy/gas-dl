 核心结论

  模型 并非完全失败：测试集 macro_RMSE=1.47、macro_MAE=1.16，H2/CH4
  R²≈0.97，整体处于可用区间。问题集中在三处：(1) 训练过早收敛后停滞 (best epoch=11/120,
  第26轮早停)；(2) 验证曲线剧烈震荡；(3) 组分预测和不为100，且小组分 MAPE 失控。

  证据与原因

1. 训练侧：欠拟合 + 不稳定
- 训练损失 epoch1=919.6 → epoch11=10.7 → epoch26=8.93，最后15轮几乎不再下降，明显进入平台期。
- 验证损失波动剧烈：epoch11=2.31，epoch12=9.94，epoch16=4.33，epoch21=13.0，单 epoch 量级跳变 3-6
  倍。
- 改善只发生在前 11 个 epoch（train_log 中 improved=True 全部集中在 1-11）。
- 触发原因（按可能性排序）：
  - LR=1e-3 + warmup 5 epoch + cosine→1e-4 对该模型偏大，warmup 结束 (epoch 6)
    后立刻进入震荡区，最优点 epoch11 LR 仍在 9.96e-4。
  - BatchNorm + batch_size=32 在波形数据上 mini-batch 统计噪声大，进一步放大震荡。
  - dropout=0.1 很低，head 仅 1 个 64 维 FC，模型容量偏小 (hidden=[32,64,64], kernel=5)，无法继续压低
    train_loss。
2. 评估侧：和约束未建模
- mean_pred_sum = 97.98 (±1.47)，平均比 100 少约 2pp；max_abs_sum_error=6.67。
- 输出层是普通 Linear→4，没有 softmax / sum=100 投影 / 约束损失，所以分量自由偏移。
- 这直接导致 H2、N2 这种小组分预测偏低（H2 预测均偏小约 2-3pp，与 train_log 一致），R²(N2)=0.89,
  R²(CO2)=0.92 明显低于 H2/CH4。
3. MAPE 异常巨大的原因
- H2 MaxAPE=57195%, N2 MaxAPE=253256%。
- 看 predictions.csv：H2 真值出现 0.54、0.78、0.18 这种接近 0 的样本，预测哪怕只差
  1pp，相对误差就被放大上千倍。
- 这不是模型坏，而是 MAPE 在零附近样本本就不可用。应改用 RMSE / MAE / R² 评估，或者只在 y_true >
  阈值时计算 MAPE（symmetric MAPE 或 ε-MAPE）。
4. 训练 vs 验证 loss 反差
- best epoch 处 train_loss=10.7, val_loss=2.3，验证比训练低 4 倍。

- 通常是：dropout/BN 训练态噪声 + train 集存在更难的样本/不同分布。需要确认 train/val/test 是否按
  mixture_id 分组——split_strategy=existing_or_group_mixture 暗示是分组划分，val 可能恰好落在更容易的
  mixture 上。这是单 seed 评估不稳定的主要来源。
  
  建议下一步（按性价比排序）
1. 加约束：输出层改 softmax * 100，或在 loss 中加 λ·|sum(pred)-100|^2。这能直接拉回 sum=100，并改善
   N2/CO2。
2. 降 LR + 增 patience：peak_lr 1e-3 → 3e-4，或 warmup 拉到 10 epoch，early_stopping_patience 15→25。
3. 增模型容量：hidden_channels 加大到 [64,128,128]，head 加一层；dropout 0.1→0.2。
4. 多 seed 验证：当前只看 seed=42，val 震荡这么大很可能是划分偶然性。跑 seed=0/1/2 看均值±std。
5. 替换/补充 MAPE：报告改用 RMSE、MAE、R²、sum_error；如仍需相对误差，对 y_true<1% 的样本单独列。
6. 检查归一化：train_loss 第一轮 919（量级远大于 val 12），确认标签是否做了归一化；如果是 raw 百分比
   (0-100)，MSE 量级合理；若 train 用了不同标量缩放则需对齐。





第 3 条「增模型容量」的含义和用途

  当前模型规格

  config.json 里：
  hidden_channels = [32, 64, 64]
  kernel_size = 5
  dropout = 0.1
  head: Linear(pooled_dim → 64) → ReLU → Dropout → Linear(64 → 4)

  参数量很小，估算只有几万到十几万级别。

  为什么判断容量不够

  证据来自 train_log.csv：

  ┌───────┬────────────┐
  │ epoch │ train_loss │
  ├───────┼────────────┤
  │ 11    │ 10.73      │
  ├───────┼────────────┤
  │ 16    │ 9.94       │
  ├───────┼────────────┤
  │ 21    │ 9.39       │
  ├───────┼────────────┤
  │ 26    │ 8.93       │
  └───────┴────────────┘

  train_loss 在 8-10 区间持续不下降，而不是降到接近 0。这叫欠拟合——模型连训练集都没拟合好，不是过拟合。

  对比：如果是过拟合，应该看到 train_loss 持续下降到 1-2，val_loss
  反而上升。现在两边都卡在高位，是典型的「模型表达能力不够」。

  改大之后做什么

- hidden_channels [32,64,64] → [64,128,128]：每层卷积通道数翻倍，能提取更丰富的波形特征。

- head 加一层：从单层 64 维 FC 改成两层（比如 pooled → 128 → 64 →
  4），让回归头有更强的非线性映射能力。

- dropout 0.1 → 0.2：容量变大后过拟合风险上升，dropout 同步提升做正则。
  
  预期收益

- train_loss 应该能从 ~9 继续往下走到 3-5 区间。

- val_loss 的下限（best）有机会从 2.3 降到 1.5 以下。

- 对 N2/CO2 这种小组分的拟合会改善（它们方差小、需要更精细的特征区分）。
  
  风险

- 参数量增加后训练慢一些（CNN1D 还不至于明显）。

- 如果 train_loss 降下去但 val
  不动，说明触到了数据本身的噪声上限，那时再考虑数据增强或多模态信息融合。
  
  判断该不该做的简单标准
  
  跑完建议 1+2 之后看 train_log：

- 如果 train_loss 还是卡在 5 以上不动 → 做建议 3。

- 如果 train_loss 已经降到 1-2 但 val 反而上升 → 不要加容量，反而要加正则或减容量。

  标签的真实分布

  x_H2:  min=0.004,  max=29.79,  mean=9.45,  std=7.74
  x_CH4: min=40.87,  max=99.11,  mean=75.50, std=10.19   <- 量级最大
  x_CO2: min=0.001,  max=14.84,  mean=5.07,  std=3.56    <- 量级最小
  x_N2:  min=0.001,  max=20.00,  mean=9.98,  std=5.79

  推断：归一化层面真正的问题

1. 标签未归一化 → MSE 偏心
   
   MSE = Σᵢ (yᵢ - ŷᵢ)²，4 个组分按方差量级贡献到总损失：
   
   ┌──────┬───────┬───────┬────────────┐
   │ 组分 │  std  │ std²  │ 占总方差比 │
   ├──────┼───────┼───────┼────────────┤
   │ CH4  │ 10.19 │ 103.8 │ 52%        │
   ├──────┼───────┼───────┼────────────┤
   │ H2   │ 7.74  │ 59.9  │ 30%        │
   ├──────┼───────┼───────┼────────────┤
   │ N2   │ 5.79  │ 33.5  │ 17%        │
   ├──────┼───────┼───────┼────────────┤
   │ CO2  │ 3.56  │ 12.7  │ 6%         │
   └──────┴───────┴───────┴────────────┘
   
   模型在反向传播时主要学 CH4，对 CO2/N2 的关注只有 6%/17%。这和 component_metrics.csv 的结果一致：
- R²(CH4)=0.975, R²(H2)=0.970（学得好）

- R²(CO2)=0.915, R²(N2)=0.893（学得差）
  
  修复方法：对 y 做 z-score 或 min-max 归一化（用 train split 统计），训练时用归一化后的 y 算
  loss，eval 时再反归一化。或者保留 raw y，loss 改成 per-component weighted MSE：loss = Σᵢ wᵢ·(yᵢ -
  ŷᵢ)²，权重 wᵢ = 1/std²ᵢ。
2. L_m 与 piston_position_m 完全重复
   
   L_m:                mean=0.8008, std=0.3699, min=0.20, max=1.40
   piston_position_m:  mean=0.8008, std=0.3699, min=0.20, max=1.40
   
   scaler 也给出相同的 mean/std。两个通道信息完全相同，slow 实际只有 7
   维有效信息。不是归一化错误，但浪费模型容量——可以删一个通道，或在物理上确认它们是否就是同一量。

3. 波形 scale_factor 跨度 60 倍
   
   ultrasonic_scale: min=4.86e-7, max=2.91e-5
   
   不同样本物理量级差 60 倍。代码里 waveform = int16 * scale 之后没有再做 per-sample 归一化，全靠
   encoder 第一层 BN 把不同样本拉齐。这在大多数情况下能
   work，但对量级差异极端的样本可能造成早期训练梯度不稳定。如果想稳一点，可以在 encoder 入口加一个
   LayerNorm（沿 sample 维）。
   
   优先级判断
   
   最值得改的是 标签归一化，预期收益最直接，能立刻拉高 N2/CO2 的 R²。
