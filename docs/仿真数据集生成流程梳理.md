# 仿真数据集生成流程梳理

> 整理日期：2026-05-19
> 适用对象：`V3.1 dual-channel waveform`
> 生成入口：`src/sim/scripts/generate_waveform_dataset.py`
> 验收入口：`src/sim/scripts/check_waveform_directionality.py`

## 1. 总览

V3.1 仿真数据集不是直接从单个物理公式一次性生成，而是由以下几层拼接而成：

1. 复用 V1 条件采样逻辑，先生成配气和工况候选条件
2. 过滤出可用于 waveform 仿真的样本
3. 为每条序列构造 `120` 步时序
4. 在每个时间步上同时生成：
   - 8 维慢变量
   - 超声对射波形
   - 光纤麦克风波形
5. 写出标签、split、metadata、scaler 和质量摘要
6. 再用单独的方向性检查脚本做物理一致性验收

可以把它理解成：

`V1 条件采样` + `V2 时序动力学` + `V3 双通道 waveform 物理模型`

## 2. 运行入口与默认参数

命令入口在 [generate_waveform_dataset.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/scripts/generate_waveform_dataset.py)：

```powershell
python src/sim/scripts/generate_waveform_dataset.py
```

主函数默认参数如下：

- `output_dir = data/waveform_v3`
- `sequence_count = 10000`
- `timesteps = 120`
- `seed = 20260514`
- `dt_s = 1.0`
- `storage = memmap`
- `noise_seed_count = 3`
- `multi_path_phase = steady`

这里有一个需要特别注意的地方：

- 现代码默认 `noise_seed_count = 3`
- 但当前仓库中的实际数据包 `data/waveform_v3/` 只有 `10000` 条序列

这说明当前数据包与现代码默认行为之间可能存在版本差异，或者实际生成时使用了不同参数。也就是说，下面梳理的是**当前代码定义的标准流程**，而不是简单等同于当前已落盘数据包的每个细节。

## 3. 顶层生成流程

入口函数 `generate_waveform_dataset(...)` 的主流程可概括为：

1. 规范化参数并建立随机数发生器
2. 创建超声和光纤麦克风通道规格对象
3. 采样四组分候选条件
4. 生成标签数组
5. 构建完整时序数组与长表
6. 按 `mixture_id` 做 train/val/test 划分
7. 基于 train split 拟合慢变量 z-score scaler
8. 写出 CSV / NPY / JSON / README 等所有数据包文件
9. 生成质量摘要

对应核心函数链路：

- `generate_waveform_dataset`
- `_waveform_sequence_condition_rows_four_component`
- `_build_waveform_sequence_arrays`
- `split_mixture_ids`
- `fit_z_score_scalers`
- `_write_waveform_metadata`
- `_waveform_quality_summary`
- `_waveform_readme`

## 4. 条件采样层

### 4.1 基础条件来自 V1 逻辑

V3 不是自己从零定义所有工况采样规则，而是先调用：

- [sim_common/conditions.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_common/conditions.py)
- [sim_common/v1_helpers.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_common/v1_helpers.py)

其中 `_condition_rows(sample_count, rng)` 会按 V1 规则生成候选条件行。

### 4.2 V1 阶段比例

V1 条件采样先按四类 stage 分配样本比例：

- `baseline_stage`: `10%`
- `distance_stage`: `40%`
- `pressure_stage`: `40%`
- `purge_stage`: `10%`

其中：

- `baseline_stage` 和 `purge_stage` 会被标记为 `calibration_control`
- V3 最终只保留 `synthetic_measurement`

所以 V3 / V2 都需要先 oversample，再过滤。

### 4.3 候选工况范围

基础条件采样的主要范围是：

- 温度 `T_C`: `15 ~ 35 °C`
- 湿度 `H_RH`: `20 ~ 80 %`
- 压力 `P_MPa`:
  - `baseline_stage`: `0.1 ~ 0.3 MPa`
  - 其他阶段: `0.1 ~ 0.709 MPa`
- 声程 `L_m`:
  - `distance_stage`: `0.2 ~ 1.8 m`
  - 其他阶段: `0.4 ~ 1.2 m`

### 4.4 `mixture_id` 的组织方式

`distance_stage` 不是完全独立采样，而是按 5 个固定声程组装：

- `0.2, 0.6, 1.0, 1.4, 1.8 m`

同一组共享一个 `mixture_id`，因此它天然形成“同一配气条件下不同声程”的成组样本。这也是后面必须按 `mixture_id` 分组切分 train/val/test 的原因。

### 4.5 四组分重采样

在 V3 中，真正被用于双通道 waveform 数据集的是：

- `build_synthetic_condition_rows_four_component(sequence_count, rng)`

它会基于 V1 候选样本重新采样四组分：

- `x_H2`
- `x_CH4`
- `x_CO2`
- `x_N2`

约束逻辑是：

- `x_H2` 参考 V1 `base_h2`，并限制在 `0 ~ 30`
- `x_CO2` 参考 V1 `base_co2`，并限制在 `0 ~ 15`
- `x_N2` 从预设范围采样
- `x_CH4 = 100 - x_H2 - x_CO2 - x_N2`
- 要求 `x_CH4` 不低于最小主体组分阈值
- 最后统一归一化到总和 `100`

因此，V3 的四组分不是直接继承 V1 行，而是“以 V1 候选为底稿，再做一次四组分重采样”。

## 5. waveform 可行性过滤

采样完四组分条件后，V3 还会做一次窗口过滤：

- `_condition_fits_waveform_window(condition, spec)`

判据是：

1. 先用 `_hidden_sound_speed_v2(...)` 计算声速
2. 计算 `peak_index = round(L_m / c_sound * sample_rate_hz)`
3. 要求主峰落在超声测量窗内

也就是说，只有满足“主峰不会跑出超声窗口”的候选条件才会进入最终数据集。

这一步是 V3 相比纯结构化时序数据更关键的约束，因为 waveform 仿真必须保证飞行时间对应的回波实际可观测。

## 6. 序列 ID、base condition 和噪声种子

条件通过过滤后，代码会进一步为每个 base condition 补充：

- `sequence_id`
- `base_condition_id`
- `noise_seed_index`
- `noise_seed`
- `_condition_seed`
- `_sequence_seed`

设计意图是：

- `condition_seed` 控制该条件的基础物理条件一致性
- `sequence_seed` 控制具体序列内的时序噪声与波形噪声
- `noise_seed_index` 允许同一 base condition 派生多个噪声重复样本

按现代码逻辑，`noise_seed_count > 1` 时，同一 base condition 会被展开成多条序列。

## 7. 数组分配与存储格式

### 7.1 默认存储格式

默认 `storage = memmap`，因此大数组会直接以 `.npy` memmap 方式落盘，而不是先全部驻留内存。

### 7.2 分配的核心数组

会预先分配以下数组：

- `ultrasonic`: `int16 [N, T, 1000]`
- `ultrasonic_scale`: `float32 [N, T]`
- `fiber_mic`: `int16 [N, T, 2000]`
- `fiber_mic_scale`: `float32 [N, T]`
- `slow`: `float32 [N, T, 8]`

其中：

- `N = len(conditions)`
- `T = timesteps`

## 8. 每条序列的时序构造

真正的核心在 `_build_waveform_sequence_arrays(...)`。

### 8.1 先生成 baseline 与 target 两套主特征

每条序列先通过 V1 的 `_generate_main_features(...)` 生成两套“单点传感器主特征”：

1. `baseline_main`
   - 配气固定为 `100% N2`
   - 表示气室基线状态
2. `target_main`
   - 配气为当前目标四组分
   - 表示目标稳态状态

这里相当于把 V1 的单点仿真结果当作：

- 时序起点
- 时序稳态终点

后续时序只是在这两者之间做动态过渡。

### 8.2 慢变量动态参数

随后调用 [sim_v2/dynamics.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_v2/dynamics.py) 中的：

- `channel_dynamic_params(sequence_rng)`

为每个通道采样一套动态参数：

- `tau_rise_system_s`
- `tau_decay_system_s`
- `noise_sigma`
- `random_walk_sigma`
- `drift_slope`

这意味着每条序列都有独立的一阶响应速度、白噪声、随机游走和线性漂移。

## 9. 单条样本内部的四阶段时序

### 9.1 阶段边界

阶段划分来自 [sim_common/phases.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_common/phases.py)。

当 `timesteps = 120` 时：

- `q1 = 20`
- `q2 = 70`
- `q3 = 100`

于是四段分别是：

- `baseline`: `0-19`
- `exposure`: `20-69`
- `steady`: `70-99`
- `recovery`: `100-119`

### 9.2 组分混合时间线

代码使用 `_phase_blend(...)` 生成 `0 ~ 1 ~ 0` 的混合系数：

- `baseline` 段：`blend = 0`
- `exposure` 段：从 `0` 逐步上升到 `1`
- `steady` 段：保持 `1`
- `recovery` 段：从 `1` 逐步回落

然后用 `_blend_composition(condition, blend)` 构造每个时间步的实际配气：

- `x_H2`, `x_CH4`, `x_CO2` 按 `blend` 从 0 过渡到目标值
- `x_N2` 从 `100` 逐步过渡到目标 `x_N2`

这相当于模拟：

- 初始纯 N2
- 通入目标混合气
- 稳态保持
- 最后再吹扫回 N2

## 10. 慢变量生成逻辑

### 10.1 动态通道

真正按时序动态变化的慢变量只有 3 个：

- `V_NDIR_CH4`
- `V_NDIR_CO2`
- `V_TCS`

它们通过 `_dynamic_slow_features(...)` 计算，核心公式为：

1. 用 `channel_value(...)` 做一阶系统响应
2. 再叠加 random walk
3. 再叠加线性 drift
4. 再叠加高斯白噪声
5. 最后做物理边界裁剪

### 10.2 环境与机械变量

以下通道不会做一阶动态仿真，而是直接从条件或路径调度得到：

- `T_C = T_C_base`
- `P_MPa = P_MPa_base`
- `H_RH = H_RH_base`
- `L_m = current_l_m`
- `piston_position_m = current_l_m`

也就是说：

- 温度、压力、湿度在单条样本内默认不随时间变化
- 声程 `L_m` 可能会因为多路径扫描策略而在部分阶段跳变

## 11. 多路径扫描逻辑

V3 额外支持 `multi_path_phase`，对应 3 种模式：

- `off`
- `baseline`
- `steady`

对应路径表：

- `WAVEFORM_PATH_LMS = (0.2, 0.6, 1.0, 1.4)`

含义是：

- `baseline` 模式：只在 baseline 段扫描不同 `L_m`
- `steady` 模式：只在 steady 段扫描不同 `L_m`
- `off` 模式：全程使用 `L_m_base`

代码通过 `_path_l_m_for_timestep(...)` 按时间步调度当前 `L_m`。

这一步会同时影响：

- 慢变量里的 `L_m`
- 超声 TOF
- 光纤麦克风直达波与反射波到达时间

## 12. 超声通道 waveform 生成

超声通道实现位于 [acoustic_waveform_v3.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/scripts/acoustic_waveform_v3.py)。

### 12.1 固定规格

- 载频：`40 kHz`
- 激励：`8-cycle burst`
- 采样率：`200 kHz`
- 测量窗：`5 ms`
- 波形长度：`1000` 点
- ADC：`int16`
- 噪声标准差：`1e-3 V`

### 12.2 生成步骤

单个时间步的超声波形按如下步骤生成：

1. 用 `_hidden_sound_speed_v2(...)` 计算声速 `c_sound`
2. 用 `_hidden_attenuation_v2(...)` 计算衰减系数 `alpha_true_npm`
3. 计算飞行时间 `tof_s = L_m / c_sound`
4. 生成 Hanning 窗的 8 周期 burst pulse
5. 把 pulse 放到 `peak_index = round(tof_s * sample_rate)` 位置
6. 幅值按 `exp(-alpha * L_m)` 衰减
7. 叠加高斯噪声
8. 以当前波形峰值做缩放，量化为 `int16`

最终返回：

- `waveform_float`
- `waveform_int16`
- `scale_factor`
- `tof_s`
- `peak_index`
- `peak_abs_v`
- `alpha_true_npm`

落盘时只保存：

- `waveform_int16`
- `scale_factor`

## 13. 光纤麦克风通道 waveform 生成

光纤麦克风实现位于 [acoustic_fiber_mic_v3.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/scripts/acoustic_fiber_mic_v3.py)。

### 13.1 固定规格

- 载频：`40 kHz`
- 激励：`8-cycle burst`
- 采样率：`200 kHz`
- 测量窗：`10 ms`
- 波形长度：`2000` 点
- `l_direct_factor = 0.5`
- `wall_reflection_coef = 0.5`
- `max_reflections = 15`
- 噪声标准差：`1e-3 V`

### 13.2 生成思路

这个通道不是单一到达峰，而是：

- 一条直达波
- 多条壁面反射回波

### 13.3 生成步骤

单个时间步的光纤麦克风波形按如下步骤生成：

1. 计算声速 `c_sound`
2. 计算衰减系数 `alpha_true_npm`
3. 计算：
   - `l_direct = L_m * l_direct_factor`
   - `tof_direct_s = l_direct / c_sound`
   - `t_round_s = 2 * L_m / c_sound`
   - `tau_s = 1 / (alpha_true_npm * c_sound)`
4. 先放置直达波
5. 再循环叠加第 `k` 次反射波：
   - 路径长度 `l_direct + 2*k*L_m`
   - 幅值 `wall_reflection_coef^k * exp(-alpha * path_length)`
   - 到达位置 `tof_direct + k * t_round`
6. 当反射幅值过小或超出窗口时停止
7. 叠加高斯噪声
8. 用峰值缩放量化成 `int16`

最终返回：

- `waveform_float`
- `waveform_int16`
- `scale_factor`
- `tof_direct_s`
- `t_round_s`
- `tau_s`
- `reflection_count`
- `reflection_amplitudes`
- `reflection_indices`

这一路的主要物理信息不再是单个 TOF，而是：

- 衰减时间常数 `tau`
- 反射峰间隔
- 反射峰数量

## 14. 主循环里每个时间步真正落盘的内容

在 `_build_waveform_sequence_arrays(...)` 的时间步循环中，代码每一步都会：

1. 计算 `phase_id`
2. 计算 `blend`
3. 生成当前 8 维慢变量
4. 生成当前组分组成 `composition`
5. 计算当前 `L_m`
6. 生成一条超声波形
7. 生成一条光纤麦克风波形
8. 写入 5 个主数组
9. 追加一行 `slow_sequence_long.csv`

也就是说，最终数据包中的每条序列实际上是：

- `120` 份超声子波形
- `120` 份光纤麦克风子波形
- `120` 个慢变量时刻

## 15. 标签生成

标签直接来自序列级条件，不随时间步变化。

代码生成方式是：

```text
labels[i] = [x_H2, x_CH4, x_CO2, x_N2]
```

因此：

- `slow` 和两路 waveform 是时序输入
- `y` 是序列级回归目标

这与后续深度学习任务是一致的，即“输入一段 120 步时序，输出一组最终气体组分”。

## 16. train / val / test 划分

划分逻辑在 [sim_common/splits.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_common/splits.py)。

规则是：

1. 对所有 `mixture_id` 去重
2. 用固定 seed 打乱
3. 按比例切分：
   - train: `70%`
   - val: `15%`
   - test: `15%`
4. 再把属于这些 `mixture_id` 的全部序列分派到对应 split

这样可以避免同一配气组的重复样本同时出现在训练和测试中。

写出的文件是：

- `splits/train_sequence_ids.csv`
- `splits/val_sequence_ids.csv`
- `splits/test_sequence_ids.csv`

## 17. scaler 生成

慢变量 scaler 的拟合逻辑在 [sim_common/scalers.py](D:/mydate/项目资料__多模态掺氢天然气/04_代码与实验/code/V3_正式实验/src/sim/sim_common/scalers.py)。

关键原则：

- 只用 `train split` 拟合
- 不使用 val/test 数据

输出两份 JSON：

1. `scaler_slow_sequence.json`
   - 全 8 通道统一记录 mean/std
2. `scaler_slow_sequence_modal.json`
   - 按模态分组记录 mean/std

分组为：

- `optical`: `V_NDIR_CH4`, `V_NDIR_CO2`
- `thermal`: `V_TCS`
- `environment`: `T_C`, `P_MPa`, `H_RH`, `L_m`, `piston_position_m`

## 20. 生成后的方向性验收

生成完成后，仓库还定义了单独的验收脚本：

```powershell
python src/sim/scripts/check_waveform_directionality.py
```

它会输出：

- `quality/waveform_directional_report.json`

### 20.1 超声通道检查项

- `L_m` 增大时，`peak_index` 是否单调增大
- `x_H2` 增大时，`peak_index` 是否单调减小
- `x_CO2` 增大时，峰值幅度是否减小、衰减系数是否增大

### 20.2 光纤麦克风检查项

- `x_CO2` 增大时，`tau` 是否减小
- 湿度增大时，`tau` 是否减小
- `x_H2` 增大时，`tau` 是否减小
- `L_m` 增大时，`t_round` 是否增大

这一步不是重新生成数据集，而是做物理方向性 sanity check。

## 22. 梳理后的核心理解

如果只保留一句话来概括这套生成逻辑，可以这样说：

> V3.1 数据集先复用 V1 规则采样四组分工况，再用 V2 的一阶动态模型把单点工况扩展成 120 步时序，随后在每个时间步上分别调用超声对射和光纤麦克风的 waveform 物理模型生成双通道声学波形，最后按 `mixture_id` 做分组切分、仅用 train 拟合慢变量 scaler，并写出完整的数组、标签、metadata 和质量摘要。

## 23. 需要特别注意的点

### 23.1 当前代码与当前数据包可能不完全同版

最明显的信号是：

- 现代码默认 `noise_seed_count = 3`
- 当前实际数据包表现为 `10000` 条序列

这意味着你在写论文或做实验复现时，最好区分：

- “当前代码定义的标准生成流程”
- “当前已落盘数据包的实际生成参数”

### 23.2 `calibration_status` 仍是 `pending`

这说明当前双通道 waveform 仍属于：

- 结构与物理关系对齐的仿真链路
- 但不是已经用真实硬件完全标定过的采集模型

### 23.3 保存的是量化原始波形，不是预提特征

数据包保留了两路原始时域波形，只是采用：

- `int16 waveform`
- `scale_factor`

的压缩保存方式。也就是说，后续模型既可以端到端读 waveform，也可以再二次提特征。
