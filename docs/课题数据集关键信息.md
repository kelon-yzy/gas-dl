> 数据版本：`V3.1 dual-channel waveform`
> 数据路径：`data/waveform_v3/`
> 说明：以下内容以仓库内当前实际数据包与生成脚本为准。

## 详细说明

### 1. 样本时序结构

- 每条样本序列包含 `120` 个时间步。
- 时间步长为 `1.0 s`，因此单条样本总时长约 `120 s`。
- `sequence_index.csv` 中的 `n_timesteps` 字段固定为 `120`，`dt_s` 固定为 `1.0`。

### 2. 声学数据结构

当前数据集包含两路独立声学波形：

1. 超声对射通道 `ultrasonic`
2. 光纤麦克风通道 `fiber_mic`

对应张量形状为：

- `ultrasonic`: `int16 [10000, 120, 1000]`
- `fiber_mic`: `int16 [10000, 120, 2000]`
- `ultrasonic_scale`: `float32 [10000, 120]`
- `fiber_mic_scale`: `float32 [10000, 120]`

含义如下：

- 第 1 维：样本数
- 第 2 维：时间步
- 第 3 维：该时间步内的波形采样点数

因此，单个时间步内：

- 超声波形长度为 `1000` 点
- 光纤麦克风波形长度为 `2000` 点

### 3. 采样率

- 两路声学通道采样率均为 `200000 Hz`
- 超声测量窗为 `0.005 s`，因此 `0.005 × 200000 = 1000` 点
- 光纤麦克风测量窗为 `0.010 s`，因此 `0.010 × 200000 = 2000` 点

慢变量部分的时间步长为 `1.0 s`，可视为低频采样率 `1 Hz`。

### 4. 慢变量通道

慢变量张量为 `slow: float32 [10000, 120, 8]`，共 `8` 个通道，顺序为：

1. `V_NDIR_CH4`
2. `V_NDIR_CO2`
3. `V_TCS`
4. `T_C`
5. `P_MPa`
6. `H_RH`
7. `L_m`
8. `piston_position_m`

### 5. 预测目标与标签形式

- 当前任务是四组分回归预测。
- 标签数组为 `y: float32 [10000, 4]`。
- 组分顺序固定为：
  - `x_H2`
  - `x_CH4`
  - `x_CO2`
  - `x_N2`
- 标签值为连续体积分数，单位可按 `vol%` 理解。
- 从 `sequence_labels.csv` 可见，每条样本对应一行四组分标签。

### 6. 数据集划分方式

当前训练/验证/测试划分不是按 `sequence_id` 随机拆分，而是：

- 先按 `mixture_id` 去重
- 再按 `mixture_id` 随机分组划分 `train / val / test`
- 默认比例约为 `0.70 / 0.15 / 0.15`

这样做的目的是避免同一配气条件的重复样本同时出现在不同 split 中，减少信息泄漏。

当前实际数据包中的 split 规模为：

| split | sequence_count | mixture_count |
| ----- | --------------:| -------------:|
| train | 7028           | 3798          |
| val   | 1478           | 814           |
| test  | 1494           | 815           |

### 7. 样本内部阶段划分

如果“数据阶段”指单个样本内部的时间阶段，则当前所有样本采用统一的 `standard_exposure` 时序，并按 `120` 步切分为 4 段：

| 阶段         | 时间步范围     | 含义           |
| ---------- | --------- | ------------ |
| `baseline` | `0-19`    | 初始基线阶段       |
| `exposure` | `20-69`   | 通入目标气体后的上升阶段 |
| `steady`   | `70-99`   | 接近稳态阶段       |
| `recovery` | `100-119` | 吹扫恢复阶段       |

### 8. 是否保留原始声学波形

当前数据包保留了两路声学波形的原始时域信息，但保存形式不是直接落盘浮点波形，而是：

- `ultrasonic_int16.npy` + `ultrasonic_scale.npy`
- `fiber_mic_int16.npy` + `fiber_mic_scale.npy`

使用时按以下规则反量化：

```text
float_wave = int16_wave × scale
```

这说明：

- 原始波形信息是保留的
- 但采用了 `int16 + scale` 的压缩存储方式
- 仓库当前未单独保存 `waveform_float.npy` 之类的原始浮点文件

## 推荐表述

可直接在论文或课题说明中使用如下表述：

> 本课题使用 V3.1 双通道 waveform 仿真数据集。每个样本由 120 个时间步组成，时间步长为 1 s。声学部分包含两路原始波形：超声对射通道每步 1000 点、光纤麦克风通道每步 2000 点，两路采样率均为 200 kHz。低频慢变量共有 8 个通道。预测目标为 4 个气体组分 `x_H2`、`x_CH4`、`x_CO2`、`x_N2`，标签采用连续回归形式。数据集按 `mixture_id` 分组划分为训练集、验证集和测试集，以避免同一配气条件跨集合泄漏。原始声学波形以 `int16 + scale` 的形式保留，可反量化恢复时域波形。
